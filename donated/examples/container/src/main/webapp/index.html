Running the configTest Sample App
  1. To test property override from file system, create folder /data/configTest/conf. Then create file configTest.global.properties in this folder. This is the overriding properties file. Put a new property there or change value for an exiting property, then wait a minute (this is the defined reload interval) then go to Trigger Reload page to trigger the refresh.
  2. Go to testing page to see all properties listed.
  3. All property files (eh-config.properties and various application configuraiton property files) are in WEB-INF/classes folder. I have a configTest.server.subscriptionServer.properties. You may want replace subscriptionServer with your WL server name if you want properties from that file to show up.
  4. Note that we have to declare a resource refrence in web.xml (or ejb-jar.xml). 
    			  
				  <resource-ref>
				    <res-ref-name>wm/configWorkManager</res-ref-name>
				    <res-type>commonj.work.WorkManager</res-type>
				    <res-auth>Container</res-auth>
				    <res-sharing-scope>Shareable</res-sharing-scope>
				  </resource-ref>
